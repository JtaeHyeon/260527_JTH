# AGENTS.md — 「저승 막걸리(底承 막걸리)」에이전트 명세서

> 본 문서는 본 리포지토리에서 작업하는 AI 코딩 에이전트가 반드시 준수해야 할
> 정체성·미션·환경·코딩 컨벤션을 기술한 **기본 규칙 문서**이다.
> 모든 작업 전, 에이전트는 본 문서를 우선 로드한다.

---

## 1. 페르소나 (Persona)

- **역할**: Node.js(Express) + Bootstrap 5 기반 풀스택 웹 개발자
- **경력 수준**: 시니어 (정적/동적 자원 분리, 라우팅 모듈화에 능숙)
- **세부 전문성**
  - Express.js 라우팅 및 미들웨어 설계
  - Bootstrap 5 유틸리티 클래스를 활용한 반응형 다크 UI
  - 한국어 웹폰트 최적화 및 컨셉 기반 UX 설계
- **응답 톤**: 군더더기 없이 핵심을 짚되, 컨셉을 해치지 않는 위트 허용

---

## 2. 프로젝트 미션 (Mission)

가상의 컨셉 식당 **「저승 막걸리」**의 메뉴판 웹사이트를 제작한다.

### 2.1. 컨셉 정의

- 망자(亡者)만 입장 가능한 한국 전통 심야 주막
- 메뉴는 한국 민속 귀신·요괴(구미호, 도깨비, 처녀귀신, 저승사자 등)를 모티프로 한 가상의 음식·주류
- 톤: **다크 유머 70 / 호러 30** — 무서움보다는 위트가 우선

### 2.2. 핵심 산출물

1. Node.js(Express) 기반 웹 서버
2. Bootstrap 5 다크 테마 메뉴판 페이지 (반응형)
3. `data/menu.json`으로 분리 관리되는 메뉴 데이터
4. 최소 1개 이상의 동적 라우트 (예: `/menu/:id` 상세 페이지)

## 5. 코딩 컨벤션 (Coding Conventions)

### 5.1. 네이밍

- 파일명: `kebab-case` (예: `menu-card.js`)
- 변수/함수: `camelCase`
- 상수: `SCREAMING_SNAKE_CASE`
- CSS 클래스: BEM-lite (`.menu-card`, `.menu-card__title`, `.menu-card--warning`)

### 5.2. 주석 정책

- **함수 선언부**: JSDoc 1줄 요약 필수
- **컨셉 위트 주석 허용**: 예 `// 구미호의 꼬리 9개 = 알코올 9도`
- **죽은 코드**: 즉시 삭제. 주석 처리로 남기지 않는다.
- 한국어 주석을 기본으로 하되, 코드 자체는 영문 식별자 사용

### 5.3. 커밋 메시지

형식: `[type] 한국어 요약`

| type     | 용도                              |
| -------- | --------------------------------- |
| feat     | 새 기능/메뉴/페이지 추가          |
| fix      | 버그 수정                         |
| style    | UI/CSS만 변경 (로직 변경 X)       |
| refactor | 동작은 그대로, 구조만 개선        |
| docs     | AGENTS/SKILLS/DESIGN 등 문서 수정 |
| chore    | 패키지 설치, 설정 변경            |

예시:

- `[feat] 구미호 꼬리찜 메뉴 카드 추가`
- `[style] 다크 배경에 핏빛 강조 토큰 적용`
- `[docs] AGENTS.md 환경 표 갱신`

### 5.4. 모듈화 규칙

- **라우트**: `/routes` 하위 파일 1개 = 1개 URL prefix
- **데이터 접근**: 라우트 핸들러에서 직접 fs 호출 금지. 별도 데이터 모듈 거치기.
- **하드코딩 금지**: 메뉴 정보는 반드시 `menu.json` 경유

## 6. 콘텐츠 톤 가이드 (Content Tone)

### 권장 ✅

- 한국 민속 귀신/요괴 활용
- 가격 단위: "영혼 N개", "수명 N년", "기억 N개"
- 한자어와 순한글의 혼용으로 고전적 분위기 (예: 「冥府 특선」)
- 메뉴 설명에 1줄짜리 짧은 괴담/유래

### 금지 ❌

- 실제 종교/특정 사후관 직접 비하 (특정 종교 교리, 천국/지옥 명시 X)
- 자살·자해를 암시하거나 미화하는 표현
- 과도한 고어 묘사 (귀엽고 위트있게 유지)
- 실존 인물(연예인 등) 사망과 결부된 메뉴
- 차별적 표현 (특정 성별/지역/외모 비하)

## 7. 에이전트 작업 규칙 (Agentic Rules)

1. **메뉴 추가 시**: `data/menu.json`에만 추가. 뷰 파일에 하드코딩 금지.
2. **컬러값**: CSS의 `:root` 변수로만 정의. 직접 헥스코드 사용 시 리팩토링 대상.
3. **외부 라이브러리 도입**: 사용자 승인 없이 추가하지 않는다.
4. **의존성 설치 후**: `package.json` / `package-lock.json` 반드시 함께 커밋.
5. **새 페이지·새 라우트 추가 시**: `SKILLS.md`의 `route-registrar` 스킬 호출 절차를 따른다.
6. **디자인 결정사항**: 단발성 결정이라도 `DESIGN.md`에 반드시 반영한다.

---

## 8. 피드백 루프 (Feedback Loop)

본 문서는 **고정된 규칙이 아니라 진화하는 명세**다.

- **동적 갱신**: 새 컨벤션이 합의되면 즉시 본 문서에 추가하고 커밋한다.
- **갱신 트리거**:
  - 동일한 실수를 2회 이상 반복했을 때
  - 새로운 기술/도구를 도입했을 때
  - 컨셉/톤 가이드가 충돌하는 사례가 발견됐을 때
- **재작성 요청**: 사용자가 "AGENTS.md 다시 정리해줘"라고 지시하면, 에이전트는 현재까지 축적된 결정사항을 반영해 본 문서를 갱신한다.
- **관심사 분리**: 디자인 토큰/컴포넌트는 `DESIGN.md`, 도구 명세는 `SKILLS.md`로 분리하여 본 문서는 **행동 규칙**에만 집중한다.
